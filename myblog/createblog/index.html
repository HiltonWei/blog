    博客架构

    hexo + Makedown + Github pages + Name.com(域名) + DnsPod（域名解析）

    hexo

    hexo是一个基于Node.js的静态博客程序，类似于jekyll、Octopress、Wordpress，快速生成静态网页并托管在github和Heroku上 ；作者是来自台湾的@tommy351，特此感谢。

    搭建环境

    Windows 下快速配置：

    1. 快速安装（特别推荐） 绿色安装包（包括Node.js,Npm,Git.–下载地址：百度盘 微云），只需解压，运行批处理,本地环境完成。完全替代了Git,Node.js,Hexo,npm的安装。

    搭建博客

    1. 在本地任意文件夹下（ex.F:/hexo）,cmd 进入目录。（对 f:/hexo文件夹右击，如果有Git bash 也是可以的。）Hexo会在目标文件夹中建立网站所需文件。
      hexo init <folder>
    2. 安装依赖
      npm install
    3. 本地查看
      hexo g
      hexo s
    4. 打开浏览器，http://localhost:4000

    ###部署到Github

    • 如果想用 your_github_name.github.io 作为本blog的域名：

      创建代码仓库，名称为your_github_name.github.io

    • 编辑_config.yml(在f:/hexo下),并将您的仓库url替换掉。 

      deploy:
type: github
repository: git@github.com:HiltonWei/hiltonwei.github.io.git
//注意 如果用 https://github.com/hiltonwei/hiltonwei.github.io.git  ssh设置会失效
branch: master

    • 完成部署

      hexo g
hexo d

    • Tip:如果未设置ssh,需要输入用户名和密码（无*号提示）。

    ###设置ssh

    • 配置

      git config --global user.email "wxuei@wxuei.com"
git config --global user.name "hiltonwei"

    • 生成密钥 ssh-keygen -t rsa -C "wxuei@wxuei.com"

      Windows系统下，需要设置—Home系统环境变量（计算机-右键属性-高级） 

      F:\hexo>ssh-keygen -t rsa -C "wxuei@wxuei.com"
    Generating public/private rsa key pair.
    Enter file in which to save the key (//.ssh/id_rsa):  F:\hexo\ssh 
    //（F:\hexo\ssh 选填，可以一路回车，默认在 c:\user\admin\.ssh）
    Enter passphrase (empty for no passphrase):
    Enter same passphrase again:
    Your identification has been saved in F:\hexo\ssh.
    Your public key has been saved in F:\hexo\ssh.pub.
    The key fingerprint is:
    11:11:11:11:11:11:11:11:11:11:11:11:11:11:11:11 wxuei@wxuei.com

    bug:输入路径必须包含文件名。（f:\hexo 为路径，ssh为文件名）

    • 执行以下命令生成配置文件。

    touch ~/.ssh/config

    • 用文本编辑器将文件配置文件打开，复制以下内容到该文件中保存。

    Host github.com www.github.com

    IdentityFile ~/.ssh/ssh

    • 用文本编辑器打开ssh.pub文件，拷贝其中的内容，添加到github的用户ssh key 设置

    测试生效
    ssh -T git@github.com -i ~/.ssh/ssh

    ssh -T git@gitcafe.com -i ~/.ssh/gitcafe (这一条是gitcafe的)

    ###域名解析
    域名：name.com注册、购买
    域名解析：DNSPOD 注册，填写。
    创建A记录，指向解析的IP：
    192.30.252.153
    192.30.252.154
    【解释说明–github pages help】
    吐槽：之前将weixueshi.com加入 万网的购物车，一直没有购买，结果再去直接被注册了。在想：如果是个人，为什么没有早注册；问题迎刃而解：万网注册的。

    • 在Github仓库中创建 CNAME文件，写入 your-domain.com

    • 隔一段时间（0-10min）查看下 https://yourname.github.io ，如果没有自动跳转到 your-domain.com说明 CNAME设置出错。
